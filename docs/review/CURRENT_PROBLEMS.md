# Current Problems

## 1. Strategic directions still behave like fixed templates
The product needs strategic directions that emerge from the current RFP's winning condition. At times, generated directions still feel like prebuilt templates attached to an RFP type.

## 2. Single-brand visitor-room RFPs can receive WDS-like logic
Pocari-style or other single-brand visitor-room RFPs sometimes receive multi-entity grouping logic similar to WDS. This can create unnecessary segmentation and weak proposal strategy.

## 3. Hydrogen / energy exhibition RFPs can receive WDS-style grouping logic
Hydrogen or energy exhibitions can be incorrectly framed through WDS-like grouping logic. These projects may need industry ecosystem, technology adoption, policy, safety, or public-value logic instead.

## 4. proposal_patterns may over-influence strategy generation
Past proposal patterns are useful references, but they can contaminate current outputs when they are treated as strategy sources rather than secondary evidence or structural inspiration.

## 5. RFP type should be a guardrail, not a fixed direction template
RFP type can help prevent mismatched outputs, but it should not determine the strategic direction by itself. The current RFP evidence should remain primary.

## 6. Strategic direction should be discovered from the current RFP's winning condition
The generator should identify what must be true to win this specific proposal: evaluator priorities, client anxiety, audience behavior, constraints, proof burden, and differentiating opportunity.

## 7. Final concept name options should appear in a separate bottom section
Final naming should be separated from strategic direction exploration. Direction cards should focus on strategy; final name options should appear later as a dedicated naming section.

## 8. Debug text and validation text should be hidden from normal cards
Internal direction debug, validation checks, and guardrail explanations are useful for development, but normal users should see polished strategy outputs by default.

## 9. Text truncation / ellipsis makes evaluation difficult
Users cannot properly critique strategy quality when important rationale, evidence, or naming explanation is hidden by truncation. Expand/collapse is better than silent ellipsis.

## 10. Outline generation still needs later review
The outline generator should be reviewed after strategy generation is stabilized. If strategy is wrong, the outline inherits the wrong logic.
